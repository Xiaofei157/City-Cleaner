4.6.2 典型 Bug 案例

Bug #001：垃圾桶诱饵无限嘲讽
严重程度：主要
问题描述：垃圾桶被摧毁后，怪物仍然朝向原位置移动
复现率：100%
原因分析：TrashCanLure 销毁前未通知 EnemyAI 清除嘲讽目标
修复方案：在 OnDestroy 中遍历附近怪物，重置 currentTarget 为玩家
修复状态：✓ 已修复（v1.0.2）
Bug #015：升级卡片图片加载失败
严重程度：次要
问题描述：偶发卡片 icon 显示为粉色问号
复现率：15%
原因分析：Resources.Load 异步加载未完成就访问 sprite
修复方案：改为预加载所有图标到缓存字典
修复状态：✓ 已修复（v1.0.3）
Bug #032：多显示器下鼠标位置偏移
严重程度：轻微
问题描述：副显示器上鼠标点击位置与实际偏差 100 像素
复现率：在多显示器环境下 100%
原因分析：Input.mousePosition 未考虑多显示器坐标系统
修复方案：使用 Camera.main.ScreenToWorldPoint 转换坐标
修复状态：⚠ 待修复（优先级：低）

5 总结与展望

5.1 项目总结

本项目成功实现了一款 2D Roguelike 割草生存游戏《城市清洁工》。游戏以城市清洁为主题，玩家扮演一名普通清洁工，使用各种清洁工具对抗由垃圾变异而成的怪物。通过引入对象池优化、三选一升级系统、多武器系统等特色玩法，为玩家提供了流畅爽快的游戏体验。

主要成果：

1. 完整的 roguelike 游戏框架，支持随机升级、永久死亡机制

2. 10 种特色武器系统，涵盖近战、远程、陷阱等多种类型

3. 8 种差异化敌人，包含普通怪、精英怪、BOSS 三种类型

4. 完善的对象池系统，性能优化效果显著

5. 数据持久化系统，支持跨局金币累积和属性成长

6. 友好的 UI 界面，包含 HUD、暂停菜单、升级界面等

5.2 技术亮点

架构设计：采用单例模式管理全局系统（GameManager、UIManager、PoolManager 等），降低了模块间耦合度。

性能优化：对象池系统使频繁创建/销毁的对象复用率提升至 95% 以上，GC 分配减少 70%。

数据驱动：使用 ScriptableObject 配置怪物和武器属性，便于策划调整和扩展内容。

模块化开发：各系统职责明确，代码注释规范，便于维护和二次开发。

5.3 不足与改进

不足之处：

1. 关卡场景单一，仅有 1 张地图

2. 武器种类偏少，缺乏被动道具

3. 敌人 AI 较简单，缺乏复杂行为模式

4. 缺少剧情和世界观设定

5. 音效和音乐资源不足

改进方向：

1. 增加多张主题地图（公园、商场、地铁站等）

2. 扩展武器库至 30+ 种，加入组合技系统

3. 引入状态机优化敌人 AI，增加包抄、伏击等战术

4. 编写背景故事，增强代入感

5. 购买或制作原创音效和 BGM

5.4 未来展望

随着 Unity 引擎技术的不断发展和游戏行业的快速变化，未来计划在以下方面继续完善：

技术层面：引入 DOTS 架构进一步提升性能，支持同屏 1000+ 单位；使用 Addressables 替代 Resources 实现更灵活的资源管理；接入 Unity Analytics 进行数据分析。

内容层面：每季度推出一个新 DLC，包含新武器、新敌人、新关卡；举办社区活动征集玩家创意；开发创意工坊支持玩家自制内容。

商业层面：登陆 Steam、WeGame 等平台；推出移动版（iOS/Android）；开发周边商品；探索 IP 衍生价值。

通过本次毕业设计的实践，不仅提升了游戏开发技术水平，更重要的是学会了如何从策划、程序、美术、测试等多个维度思考和完善一款游戏。这些经验将为未来的职业发展奠定坚实基础。
