

### 🎯 **一、核心管理器 (Managers)**

#### 1. **GameManager** - 游戏核心管理器 ⭐⭐
- **作用**: 游戏的总控制器，使用单例模式
- **职责**: 
  - 确保场景中只有一个全局实例
  - 跨场景持久化（DontDestroyOnLoad）
  - 管理全局游戏数据（GameData）
  - 游戏整体流程控制
- **数据结构**:
  ```csharp
  public class GameData {
      public int gold; // 金币数据
      // 预留：角色 hp、攻击、移动速度等
  }
  ```
- **核心方法**:
  - `SaveGame()`: 保存游戏数据到 JSON 文件
- **依赖**: 
  - `JsonManager` (数据持久化)
- **关联**: 无直接依赖，其他管理器独立运行
- **生命周期**: Awake 加载数据 → Start 准备开始 → 持续运行

#### 2. **UIManager** - UI 显示管理器 ⭐⭐⭐
- **作用**: 统一管理所有 UI 元素的显示和更新
- **核心方法**:
  - `UpdateHealth(current, max)`: 更新血条 + 生命值数值文本（如 80/100）
  - `UpdateLevel(lvl)`: 更新等级显示（Lv.X）
  - `UpdateXP(current, next)`: 更新经验条进度 + 经验数值（如 15/50）
  - `UpdateGold(gold)`: 更新金币显示
  - `UpdateWeaponSlots(weapons)`: 更新武器槽位图标（最多 4 个）
- **UI 组件**:
  - Slider: healthBar, xpBar
  - Text: levelText, healthText, xpText, goldText
  - Image[]: weaponSlots (4 个)
- **依赖**: 
  - 需要 UI 元素引用（Slider、Text、Image）
  - 被 `PlayerStats` 调用
  - `SceneManager.sceneLoaded` 事件监听
- **数据流向**: PlayerStats → UIManager → UI 显示
- **特殊功能**: ESC 键打开暂停面板

#### 3. **LevelUpManager** - 升级系统管理器 ⭐⭐⭐
- **作用**: 处理玩家升级和三选一卡片系统
- **核心方法**:
  - `ShowLevelUpUI()`: 暂停游戏，生成 3 张随机卡片
  - `ApplyUpgrade(buff)`: 应用属性卡效果（HP/攻速/移速）
  - `ApplyNewWeapon(weapon)`: 装备新武器
  - `InitializeCard(ui, index, pWeapon)`: 智能生成卡片逻辑
  - `GetRandomNewWeapon(pWeapon)`: 从武器池排除已拥有武器后随机
- **依赖**:
  - `UpgradeCard_SO` (升级卡片数据)
  - `WeaponData_SO` (武器数据)
  - `PlayerStats` (应用升级效果)
  - `PlayerWeaponManager` (检查已有武器)
- **卡片生成规则**:
  - 如果武器栏未满（<4），第一张卡优先出武器
  - 否则随机生成普通 Buff 卡
- **数据流**: PlayerStats.LevelUp → LevelUpManager → 生成卡片 → 玩家选择 → 应用效果

#### 4. **PoolManager** - 对象池管理器 ⭐⭐
- **作用**: 优化性能，复用对象避免频繁实例化/销毁
- **数据结构**: `Dictionary<string, Queue<GameObject>>` 按名称分类管理
- **核心方法**:
  - `Get(prefab)`: 从池中获取对象，没有则创建新的
  - `Return(obj)`: 将对象放回池中（SetActive(false)）
  - `ClearAllPools()`: 清空所有池（场景切换时调用）
- **使用场景**:
  - 敌人生成/死亡
  - 子弹生成/消失
  - 经验球生成/拾取
  - 伤害飘字显示
- **优势**: 减少 GC（垃圾回收），提升性能，避免卡顿

#### 5. **GameLevelManager** - 关卡管理器 ⭐⭐
- **作用**: 管理游戏进度和怪物生成
- **核心数据**:
  - `currentStageLevel`: 当前关卡
  - `gameTimer`: 游戏时间计时器
  - `spawnInterval`: 刷怪间隔
- **核心方法**:
  - `TrySpawnEnemy()`: 生成普通怪物
  - `SpawnBoss()`: 生成 BOSS（300 秒时触发）
  - `SpawnElite()`: 生成精英怪（每 60 秒）
  - `SpawnAtEdge(data, isImportant)`: 在相机视野边缘生成
- **依赖**:
  - `EnemyData_SO` (怪物配置)
  - `PoolManager` (对象池)
- **生成规则**: 根据关卡限制（minLevel/maxLevel）筛选可生成的怪物

#### 6. **ResourcesManager** - 资源管理器 ⭐
- **作用**: 统一管理和缓存 Resources 文件夹中的预制体
- **数据结构**: `Dictionary<string, GameObject>` 缓存常用预制体
- **核心方法**:
  - `LoadPrefab(name)`: 从缓存或 Resources 加载预制体
  - `InstantiatePrefab(name, parent, pos, rot)`: 实例化预制体
  - `LoadSprite(name)`: 加载精灵图片
  - `ClearCache()`: 清空缓存
- **优势**: 避免重复加载，提升资源访问效率

### 👤 **二、玩家系统 (Player)**

#### 1. **PlayerStats** - 玩家属性数据中心 ⭐⭐⭐
- **作用**: 管理玩家所有核心属性
- **核心数据**:
  - 生命值：`maxHealth`, `currentHealth`
  - 移动速度：`moveSpeed`
  - 战斗加成：`baseAttackBonus`, `attackSpeedBonus`
  - 升级系统：`currentLevel`, `currentXP`, `xpToNextLevel`
  - **金币系统**: `sessionGold` (本局金币), `totalGold` (永久金币)
- **核心方法**:
  - `TakeDamage(damage)`: 受伤并更新 UI 血条
  - `AddXP(amount)`: 获得经验并更新 UI 经验条
  - `LevelUp()`: 升级并触发升级 UI
  - `AddGold(amount)`: 获得金币并保存
  - `SpendGold(amount)`: 消耗金币（优先消耗永久金币）
  - `GetFinalDamage(baseDamage)`: 计算最终伤害（武器基础伤害 * (1 + 攻击力加成%))
  - `GetAttackSpeedMultiplier()`: 获取攻速倍率 (1 + 攻速加成%)
- **依赖**: 
  - 强依赖 `UIManager` 进行 UI 更新
  - 触发 `LevelUpManager` 显示升级界面
  - 依赖 `GameManager` 保存金币数据
- **关键逻辑**: 所有属性变化都会立即同步到 UI
- **金币机制**: 
  - 每局游戏重置 sessionGold
  - totalGold = 存档金币 + sessionGold
  - 死亡时自动累加到存档

#### 2. **PlayerController** - 玩家控制器
- **作用**: 处理玩家移动输入
- **核心方法**:
  - `Update()`: 获取 WASD/方向键输入
  - `FixedUpdate()`: 通过 Rigidbody2D 实现物理移动
- **依赖**: 
  - `PlayerStats` (获取移动速度)
  - `Rigidbody2D` (物理组件)
- **特点**: 使用 normalized 保证斜向移动不会更快

#### 3. **PlayerWeaponManager** - 玩家武器管理器 ⭐
- **作用**: 管理玩家已装备的武器（最多 4 个）
- **核心方法**:
  - `EquipWeapon()`: 装备新武器并通知 UI 更新
  - 初始化时自动装备起始武器
- **依赖**:
  - `WeaponData_SO` (武器配置数据)
  - `WeaponBase` (武器基类)
  - `UIManager` (更新武器槽位显示)
- **数据流**: 获得武器 → 实例化 → 初始化 → 添加到列表 → 更新 UI

#### 4. **ExperienceOrb** - 经验球
- **作用**: 可拾取的经验值道具
- **核心方法**:
  - `OnTriggerEnter2D()`: 检测玩家触碰，给予经验后返回对象池
- **依赖**:
  - `PlayerStats` (添加经验)
  - `PoolManager` (对象池回收)
- **生成方式**: EnemyAI.Die() → DropXP() → PoolManager.Get()

### 🔫 **三、武器系统 (Weapons)**

#### 1. **WeaponBase** - 武器基类 ⭐⭐
- **作用**: 所有武器的抽象基类，定义通用逻辑
- **核心机制**:
  - `Initialize()`: 动态注入武器数据和玩家引用
  - `Update()`: 自动按攻速冷却调用 Attack()
  - `CalculateCooldown()`: 根据基础攻速和玩家攻速加成计算实际冷却
- **设计模式**: 模板方法模式
- **安全机制**: isInitialized 开关防止未初始化就运行
- **子类需实现**: `Attack()` 抽象方法

#### 2. **具体武器实现** (继承自 WeaponBase):

**VacuumWeapon (吸尘器)**:
- 特性：范围吸附 + 持续伤害
- `PullEnemies()`: 每帧拉扯范围内敌人
- `Attack()`: 周期性对范围内所有敌人造成伤害
- **配置参数**: effectRadius=4f, pullSpeed=2f

**WaterGunWeapon (水枪)**:
- 特性：远程投射物
- 使用 `WaterBullet` 预制体
- `GetNearestEnemy()`: 寻找最近敌人（范围 10f）
- **攻击流程**: 锁敌 → 计算角度 → 发射子弹 → 初始化伤害

**LawnmowerWeapon (割草机)**:
- 特性：近战旋转刀片
- 使用 `LawnmowerBlade` 旋转攻击
-  blades 围绕玩家旋转造成持续伤害

**TongsWeapon (垃圾钳)**:
- 特性：远程夹取 + 范围伤害
- `clampRadius=1.5f`: 夹取范围半径
- `attackRange=5f`: 索敌范围
- **特殊效果**: 被夹取的敌人播放特殊动画
- **攻击流程**: 锁定目标 → 瞬移到目标位置 → 范围夹击

**SprayWeapon (喷雾器)**:
- 特性：扇形范围攻击
- 使用 `OverlapCircleAll` 检测锥形区域

**SnowplowWeapon (推雪铲/扫帚)**:
- 特性：前方推进攻击
- 生成 `SnowplowMinion` 跟随物
- minion 在玩家前方持续推动并造成伤害

**TrashCanWeapon (垃圾桶)**:
- 特性：放置型防御塔
- 使用 `TrashCanLure` 吸引敌人
- **嘲讽机制**: 附近 10f 范围内的敌人优先攻击垃圾桶
- **持续时间**: 30 秒后自动销毁

### 👾 **四、敌人系统 (Enemies)**

#### **EnemyAI** - 敌人人工智能 ⭐⭐⭐
- **作用**: 控制敌人行为（追踪、攻击、死亡）
- **核心数据**:
  - `currentHP`: 当前生命值
  - `currentTarget`: 当前攻击目标（玩家或垃圾桶）
  - `animator`, `spriteRenderer`, `rb`: 组件引用
- **核心方法**:
  - `Initialize(data)`: 从对象池取出时初始化数据 + 血条
  - `Update()`: 追踪玩家移动 + 翻转朝向
  - `FixedUpdate()`: 物理移动向目标靠近
  - `OnCollisionStay2D()`: 碰撞时对玩家/垃圾桶造成持续伤害
  - `TakeDamage(damage)`: 承受伤害 + 更新血条 + 显示飘字
  - `CheckTaunt()`: 检查附近的垃圾桶诱饵，切换目标
  - `OnTongsClamped()`: 被垃圾钳夹取时播放特效
  - `Die()`: 死亡时掉落经验球和金币
- **依赖**:
  - `EnemyData_SO` (敌人配置数据)
  - `PlayerStats` (对玩家造成伤害)
  - `PoolManager` (死亡后回收)
  - `ExperienceOrb` (生成经验球)
  - `TrashCanLure` (嘲讽目标)
- **血条显示模式**:
  - AlwaysOn: 一直显示
  - ShowOnHit: 受击时显示 2 秒
  - AlwaysOff: 一直不显示
- **数据流**: 生成 → 追踪 → 攻击 → 受伤 → 死亡 → 掉落经验 + 金币

### 🃏 **五、UI 系统**

#### **UpgradeCardUI** - 升级卡片 UI
- **作用**: 显示单个升级卡片的内容
- **核心方法**:
  - `Setup(cardData)`: 显示属性卡（名称 + 图标 + 描述）
  - `SetupWeapon(weaponData)`: 显示武器卡（名称 + 图标 + "新武器:" 前缀）
  - `OnClick()`: 点击时应用升级效果并关闭 UI
- **依赖**:
  - `UpgradeCard_SO` / `WeaponData_SO` (显示内容)
  - `LevelUpManager` (应用效果)
- **组成**: Text(名称) + Image(图标) + Text(描述) + Button(点击)

#### **DamageText** - 伤害飘字 ⭐
- **作用**: 在敌人头顶显示伤害数字并向上飘动
- **核心方法**:
  - `Setup(damage)`: 初始化伤害值、颜色、随机偏移
  - `Update()`: 向上移动 + 逐渐透明 + 寿命结束回收
- **配置参数**: 
  - moveSpeed=2f (向上速度)
  - fadeSpeed=2f (透明速度)
  - lifeTime=1f (存活时间)
- **依赖**: `PoolManager` (对象池回收)

#### **GameOverPanel** - 游戏结束面板
- **作用**: 显示结算界面
- **功能**: 
  - 显示本局获得金币
  - 返回主菜单按钮
  - 重新开始按钮

### 📦 **六、ScriptableObject 数据类 (Resources)**

#### 1. **WeaponData_SO** - 武器数据资产 ⭐
- **作用**: 可序列化的武器配置数据
- **字段**:
  - 基本信息：weaponName, icon, description
  - 战斗属性：prefab, baseDamage, baseFireRate
- **优势**: 在 Project 窗口右键创建资产，无需写代码即可配置武器

#### 2. **UpgradeCard_SO** - 升级卡片数据资产 ⭐
- **作用**: 可序列化的升级卡片配置
- **字段**:
  - 卡片信息：cardName, icon, description
  - 奖励规则：rarity, canAppearConsecutively
  - 属性提升：statType, valueType, value
- **枚举**:
  - CardRarity: White(白卡), Blue(蓝卡)
  - StatType: HP/AttackSpeed/MoveSpeed/Damage
  - ValueType: Numeric(数值)/Percentage(百分比)

#### 3. **EnemyData_SO** - 敌人数据资产 ⭐⭐
- **作用**: 敌人配置数据，可在 Inspector 中创建
- **字段**:
  - 基本信息：enemyName, prefab, maxHP, moveSpeed
  - 掉落设置：xpDrop, goldDropChance, goldDropMin, goldDropMax
  - 生成规则：minLevelAppearance, maxLevelAppearance
  - 怪物类型：isBoss, isElite, specificLevels[]
- **示例数据**:
  ```yaml
  BrokenBot_Data:
    enemyName: "最终 BOSS：故障机器人"
    maxHP: 1000
    xpDrop: 50
    goldDropChance: 0.939
    isBoss: true
    minLevelAppearance: 10
  ```

### 🎨 **八、关键设计模式**

1. **单例模式**: GameManager, UIManager, LevelUpManager, PoolManager
2. **模板方法模式**: WeaponBase 定义框架，子类实现 Attack()
3. **工厂模式**: PoolManager.Get() 创建/复用对象
4. **ScriptableObject 架构**: 数据与逻辑分离，便于策划配置
5. **观察者模式**: PlayerStats 变化 → UIManager 更新
